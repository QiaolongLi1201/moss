# 发布与演示手册（Release & Demo Runbook）

适用：本次大型活动演示 + 版本发布。目标：黄金路径弹无虚发，现场任何故障都有预案。

## 本版本亮点（发布说明素材）

- 验证式连接：`/connect` 先探测 SSH 再宣告成功；失败按原因分类（认证/不可达/端口/解析），中文引导 + 自动预填重试命令，补密码回车即可。
- 板卡模式：连接后 `exec`/`read_file`/`write_file`/`edit_file` 等默认工具直接在板卡执行，整个会话如同在板卡上运行 moss；`/disconnect` 或空输入 Ctrl+D 一键回本机，工具按引用精确恢复；`/status` 与状态栏显示 BOARD 标识；`--hybrid` 保留旧行为。
- 根因修复：SSH 命令整体被引号包裹的缺陷修复——此前所有多词远程命令（device_exec、诊断、ros2 全family）实际不可用；附带 sshd 语义的端到端回归测试。
- 诚实工程一致化：`ros2_launch` 验证进程存活后才报 Launched；SSH 失败一律 THROW（skill 证据不再把失败记成成功）；写入按字节数校验。
- 渲染修复：中文长句不再被强插空格断词；列表双符号（`* •`）修复，有序/嵌套列表正确。
- 附件加固：图片按魔数校验（空文件/伪造扩展名拒收并警告），真实签名优先于扩展名。
- Skill 质量闸门：低置信候选 promote 需 `--force`；蒸馏稿不再硬编码板卡 IP；`processSession` 的完成状态改为调用方契约。
- 命令注册表：REPL/TUI 共享分发（12 命令 + 别名），未知命令统一中文提示，告别双链漂移。

## 演示前检查清单

前一天：

- [ ] `npm run build && npm run verify` 全绿（本仓库当前 138 agent + 10 skills spec）。
- [ ] 板卡固定 IP（或随身热点固定网段）；手动 `ssh sunrise@<ip>` 验证一次凭证。
- [ ] 强烈建议 `ssh-copy-id sunrise@<ip>` 配免密——演示最稳路径；备用密码方案确认本机已装 `sshpass`。
- [ ] `export LANG=zh_CN.UTF-8`（启用中文引导与提示）。
- [ ] 演示工作区 `git status` 干净；杀掉所有旧 moss 进程（旧构建的内存幽灵会产生陈旧输出）。

开场前 10 分钟：

- [ ] 终端 ≥ 120 列、字号调到后排可见；
- [ ] `ping <板卡ip>` 一次；
- [ ] 启动 moss，确认欢迎卡与 `/status` 正常。

## 演示脚本（黄金路径，约 8 分钟）

1. 启动 `moss` → 欢迎卡（cwd/model 一目了然）。
2. `/status` → 展示 board: not connected。
3. **卖点一（诚实工程）**：`/connect sunrise@<ip>` 故意不带密码 → 现场展示中文失败分类与"已为你预填"的重试命令 → 补上密码回车。
4. 出现 `verified, remote hostname: …` + 板卡模式提示 → 再看 `/status`：`BOARD MODE (default tools run on the board)`。
5. **卖点二（板卡即工作区）**：自然语言下达"看看板卡上有哪些 ROS 话题，相机节点在不在"→ 工具直接在板上执行。
6. "把 /etc/xxx.conf 里的 A 改成 B" → edit_file 在板卡原子写入 + 字节校验回执。
7. **卖点三（一键回家）**：空输入按 Ctrl+D → 断开提示 + 本地工具恢复 → 随手跑一条本机命令对比。
8. `/skills` → 展示本次会话自动沉淀的技能候选（学习闭环）。

## 现场故障预案

- 板卡连不上：失败信息会直接给出原因与下一步；最后手段 `/connect <ip> --no-verify`（向观众坦言"跳过探测"），或切换到预录终端回放。
- sshpass 缺失/密码弹错：错误信息自带安装与凭证指引；按提示走即可，这本身就是卖点。
- 投影窄屏：中文断词已修复；仍建议 ≥120 列避免表格换行。
- 输出异常/陈旧：第一反应是重启 moss 进程（内存里可能是旧构建）。

- 注册表 phase 2b（helper 抽离 tui.ts）、自定义命令（.moss/commands/）等一律活动后进行。

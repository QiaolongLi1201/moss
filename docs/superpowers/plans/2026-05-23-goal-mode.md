# Goal Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a host-neutral thread goal mode to `@rdk-moss/agent` so hosts can set, view, pause, resume, complete, block, and clear a session goal without introducing background autonomous execution.

**Architecture:** Store goal state as an internal checkpoint message in the existing `SessionStore` message stream, matching the current task-frame checkpoint pattern and avoiding any required `SessionStore` interface change. Inject active or paused goal context into `DmossAgent.buildSystemPrompt()` during `streamChat()`, while leaving CLI slash command parsing and UI controls to host applications.

**Tech Stack:** TypeScript ESM in `packages/dmoss-agent/src/core`, Node `assert` `.spec.mjs` tests against built `dist`, existing `InMemorySessionStore` and `JsonlSessionStore` replay behavior.

**Status:** Implemented and pushed to `main` in commit `597f3e2`.
